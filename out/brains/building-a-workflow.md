# 🧠 Building a Workflow / Construcción de un Flujo de Trabajo

> Cerebro documental — el Handbook es la columna; la investigación es aditiva y citada. / *Documentary brain — Handbook is the backbone; research is additive and cited.*

## English

Building a workflow for remote sensing crop mapping requires a systematic approach that integrates data acquisition, preprocessing, analysis, and validation into a reproducible pipeline. A canonical workflow composites Sentinel-2 L2A imagery (e.g., 10-daily median composites) with Sentinel-1 sigma0 backscatter, masks agricultural areas using land cover products like WorldCover, aligns all data to a common grid, and fuses them into a single datacube for classification [R1][R3]. This workflow is increasingly expressed as technology-agnostic openEO 'process graphs' that run on cloud back-ends such as the Copernicus Data Space Ecosystem, reducing code complexity and enabling open-science reproducibility [R2]. The ESA WorldCereal system provides a global, seasonal, and reproducible crop and irrigation mapping framework built on this stack [R4][R7][R8]. Key distinctions include the shift from ad-hoc scripting to standardized, reusable process graphs; the emphasis on multi-sensor fusion and temporal composites to handle cloud cover and data gaps; and the integration of reference data modules for in-situ quality and calibration [R9]. Grounding in Handbook concepts: **methodology** ensures reproducibility through systematic principles; **process** translates these principles into a sequence of steps; **system** packages methods and tools into a structured, scalable solution; **step** defines each operation (e.g., compositing, masking, fusion) that must be executed in order. The gap filled is the practical implementation of a modular, cloud-native workflow that moves from theoretical guidelines to operational, consistent crop mapping at continental or global scales, as demonstrated by WorldCereal [R5][R6].

## Español

Construir un flujo de trabajo para la cartografía de cultivos mediante teledetección requiere un enfoque sistemático que integre adquisición de datos, preprocesamiento, análisis y validación en un pipeline reproducible. Un flujo de trabajo canónico compone imágenes Sentinel-2 L2A (por ejemplo, compuestos medianos cada 10 días) con retrodispersión sigma0 de Sentinel-1, enmascara áreas agrícolas utilizando productos de cobertura terrestre como WorldCover, alinea todos los datos a una cuadrícula común y los fusiona en un solo datacube para clasificación [R1][R3]. Este flujo de trabajo se expresa cada vez más como 'gráficos de proceso' de openEO independientes de la tecnología, que se ejecutan en backends en la nube como el Copernicus Data Space Ecosystem, reduciendo la complejidad del código y permitiendo la reproducibilidad de ciencia abierta [R2]. El sistema ESA WorldCereal proporciona un marco global, estacional y reproducible de cartografía de cultivos y riego construido sobre esta pila [R4][R7][R8]. Las distinciones clave incluyen el cambio de scripts ad hoc a gráficos de proceso estandarizados y reutilizables; el énfasis en la fusión multi-sensor y compuestos temporales para manejar nubes y vacíos de datos; y la integración de módulos de datos de referencia para calidad in-situ y calibración [R9]. Fundamentación en conceptos del Manual: **methodology** asegura reproducibilidad mediante principios sistemáticos; **process** traduce estos principios a una secuencia de pasos; **system** empaqueta métodos y herramientas en una solución estructurada y escalable; **step** define cada operación (ej. composición, enmascaramiento, fusión) que debe ejecutarse en orden. El vacío que llena es la implementación práctica de un flujo de trabajo modular y nativo de la nube que pasa de pautas teóricas a cartografía operativa y consistente a escalas continentales o globales, como demuestra WorldCereal [R5][R6].
