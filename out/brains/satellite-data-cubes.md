# 🧠 Satellite Data Cubes / Cubos de Datos Satelitales

> Cerebro documental — el Handbook es la columna; la investigación es aditiva y citada. / *Documentary brain — Handbook is the backbone; research is additive and cited.*

## English

Satellite Data Cubes represent a paradigm shift in Earth Observation (EO) data management, transforming massive archives of satellite imagery into structured, multi-dimensional arrays indexed by spatial coordinates (x, y) and time (t). This framework delivers Analysis-Ready Data (ARD) — imagery that has undergone geometric, radiometric, and atmospheric correction, thus eliminating the need for users to perform complex pre-processing. The Open Data Cube (ODC) is the open-source standard for building and operating such data cubes, enabling efficient storage, query, and analysis of petabyte-scale EO archives [R1]. For agricultural monitoring, the Agriculture monitoring Data Cube (ADC) automates the indexing of optical and Synthetic Aperture Radar (SAR) imagery into a ready-to-use pipeline [R2]. A key methodological distinction is the 'time-first, space-later' paradigm implemented in the SITS (Satellite Image Time Series) framework, which organizes big EO images as data cubes and prioritizes temporal analysis over spatial extent — ideal for land and crop classification from dense time series [R3]. This approach optimizes machine learning workflows and reduces computational overhead. Data cubes are accessible via cloud-based services like the Copernicus Data Space Ecosystem, which provides harmonized Sentinel imagery as ARD [R4]. Grounding in Handbook concepts: **spatial-datasets**: Data cubes are the premier example of spatial datasets, offering consistent, georeferenced time series. **satellite-data-cube**: This concept is the core architecture; cubes are constructed from satellite-image-collections and deliver ARD for time-series analysis. **copernicus-data-space-ecosystem**: Data cubes rely on cloud platforms like Copernicus to provide scalable access to big EO data. **dataset**: A satellite data cube is a complex dataset that integrates imagery, metadata, and analysis tools. **cloud-based-services**: Data cubes are typically hosted on cloud infrastructure, enabling on-demand processing. **satellite-image-collections**: These serve as the raw input; data cubes reorganize them into analysis-ready arrays. The gap filled: Traditional remote sensing workflows involved downloading scenes, performing manual corrections, and stacking bands — a tedious, error-prone process. Data cubes pre-compute ARD and allow analysts to focus on extracting agricultural statistics, such as crop area estimation, phenology monitoring, and yield prediction, using consistent time series across large regions. The framework supports high-dimensional compositing and statistics (e.g., using pixel composites) for robust mapping in fragmented landscapes [R5][R6][R7][R8][R9].

## Español

Los Cubos de Datos Satelitales representan un cambio de paradigma en la gestión de datos de Observación de la Tierra (EO), transformando enormes archivos de imágenes satelitales en arreglos multidimensionales estructurados indexados por coordenadas espaciales (x, y) y tiempo (t). Este marco entrega Datos Listos para Análisis (ARD) — imágenes que han sido corregidas geométrica, radiométrica y atmosféricamente, eliminando la necesidad de que los usuarios realicen un preprocesamiento complejo. El Open Data Cube (ODC) es el estándar de código abierto para construir y operar estos cubos de datos, permitiendo el almacenamiento, consulta y análisis eficiente de archivos EO a escala de petabytes [R1]. Para el monitoreo agrícola, el Agriculture monitoring Data Cube (ADC) automatiza la indexación de imágenes ópticas y de Radar de Apertura Sintética (SAR) en un pipeline listo para usar [R2]. Una distinción metodológica clave es el paradigma 'primero el tiempo, luego el espacio' implementado en el marco SITS (Satellite Image Time Series), que organiza imágenes EO grandes como cubos de datos y prioriza el análisis temporal sobre la extensión espacial — ideal para clasificación de cultivos y uso del suelo a partir de series temporales densas [R3]. Este enfoque optimiza los flujos de trabajo de aprendizaje automático y reduce la carga computacional. Los cubos de datos son accesibles a través de servicios en la nube como el Copernicus Data Space Ecosystem, que proporciona imágenes Sentinel armonizadas como ARD [R4]. Conexión con conceptos del Manual: **spatial-datasets**: Los cubos de datos son el ejemplo principal de datasets espaciales, ofreciendo series temporales georreferenciadas consistentes. **satellite-data-cube**: Este concepto es la arquitectura central; los cubos se construyen a partir de colecciones de imágenes satelitales y entregan ARD para análisis de series temporales. **copernicus-data-space-ecosystem**: Los cubos de datos dependen de plataformas en la nube como Copernicus para proporcionar acceso escalable a grandes datos EO. **dataset**: Un cubo de datos satelitales es un dataset complejo que integra imágenes, metadatos y herramientas de análisis. **cloud-based-services**: Los cubos de datos suelen alojarse en infraestructura en la nube, permitiendo procesamiento bajo demanda. **satellite-image-collections**: Estas sirven como entrada bruta; los cubos las reorganizan en arreglos listos para análisis. El vacío que llenan: Los flujos de trabajo tradicionales de teledetección implicaban descargar escenas, realizar correcciones manuales y apilar bandas — un proceso tedioso y propenso a errores. Los cubos de datos precomputan ARD y permiten a los analistas concentrarse en extraer estadísticas agrícolas, como estimación de área de cultivo, monitoreo de fenología y predicción de rendimiento, utilizando series temporales consistentes en grandes regiones. El marco apoya composición y estadísticas de alta dimensión (por ejemplo, usando composites de píxeles) para mapeos robustos en paisajes fragmentados [R5][R6][R7][R8][R9].
